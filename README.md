## Presto

本项目的目的为开发一个“[slides.com](https://slides.com) 的替代品”，这是一款精简轻量级的应用，使用起来更加轻松有趣，并将“在未来几十年彻底改变演示行业”。你以基于Web的应用程序的形式提出了这个解决方案，并将这个测验应用程序命名为🪄🪄🪄**Presto**🪄🪄🪄。

### 2.1. 功能集 1. 登录和演示创建

此功能集仅关注注册、登录和注销功能。它不涉及登录后的任何功能或屏幕 - 如果登录后的仪表板只是一个带有注销按钮的空白屏幕，那么对于此功能集而言，这已经足够了。

### 2.1.0 登陆页面

- 此屏幕必须存在唯一的路由，例如 `/`
- 登陆页面必须是一个简单的欢迎页面，包含登录/注册条目

#### 2.1.1. 登录屏幕

- 此屏幕必须存在唯一的路由，例如 `/` `/login`
- 用户必须能够在表单中输入他们的 `email` 和 `password`
- 必须有一个按钮用于提交表单
- 如果用户尝试登录时表单提交失败，则应显示合理的错误消息
- 登录表单必须能够通过 `enter` 键提交，除了点击按钮之外，按 `enter` 键登录也应是另一种选择
- 登录成功后，用户将进入仪表板屏幕

#### 2.1.2. 注册屏幕

- 此屏幕必须存在唯一的路由
- 用户必须能够在表单中输入他们的 `email`、`password` 和 `name`
- 应存在一个确认 `password` 字段，供用户重新输入密码
- 如果两个密码不匹配，用户应在提交前收到错误消息。
- 如果用户尝试注册时表单提交失败，应显示合理的错误消息。
- 必须存在允许提交表单的按钮。
- 注册表单必须能够通过 Enter 键提交，除了点击按钮之外，还应提供按 Enter 键注册的替代选项。
- 注册成功后，用户将进入仪表板屏幕。

#### 2.1.3. 注销按钮

- 所有需要授权用户的屏幕上都应存在注销按钮。
- 点击此注销按钮后，将返回到登录页面。

### 2.2. 功能集 2. 设置幻灯片

#### 2.2.0. 仪表板页面

- 此屏幕必须存在唯一的路径，例如 `/dashboard`

#### 2.2.1. 仪表板上的新演示文稿

- 登录后，用户应看到一个仪表板，其中包含一个仅在仪表板上可见的按钮，用于创建新演示文稿。例如“新演示文稿”。
- 按下此按钮后，将出现一个弹出窗口（[modal](https://www.w3schools.com/w3css/w3css_modal.asp)/[dialog](https://m2.material.io/components/dialogs)），用户可以在其中输入新演示文稿的“名称”、“描述”并添加缩略图。
- 此弹出窗口应包含一个“创建”按钮，供用户点击并创建演示文稿。用户点击“创建”按钮后，弹出窗口应消失，新的演示文稿创建完成并显示在仪表板上。这是一个默认的演示文稿，包含一张空幻灯片（稍后会详细介绍）。

#### 2.2.2. 仪表板上的演示文稿列表

- 在仪表板页面上，每个演示文稿的[卡片](https://m2.material.io/components/cards) 应显示为宽高比为 **2:1** 的矩形。
- 每个矩形应包含“名称”、“缩略图”（如果为空则显示为灰色方块）、“描述”（如果为空则无文本）以及其包含的幻灯片数量。
- 如果需要，矩形应均匀分布在多行和多列中，每个矩形的宽度至少为 100px，矩形在不同视口中的实际宽度应看起来合理。

#### 2.2.3. 演示文稿控件基础知识

- 当点击仪表板上的某个演示文稿矩形时，用户应被引导至一个由演示文稿 ID 参数化的新的唯一路径，该路径始终加载幻灯片组中的第一张幻灯片。此路径用于编辑特定演示文稿，用户可以在此页面内添加/删除/编辑（稍后会详细介绍）此演示文稿中的幻灯片。
- 无论用户当前位于哪张幻灯片上，两个关键控件都应始终可见且功能正常：
- “返回”按钮用于将用户返回到仪表盘。
- “删除演示文稿”按钮会触发一个包含“您确定吗？”文本的弹出窗口。如果点击“是”，则演示文稿将被删除，用户将转到仪表盘页面。如果点击“否”，则弹出窗口将消失，页面保持不变。

#### 2.2.4. 标题和缩略图编辑

- 查看特定演示文稿时，无论用户当前位于哪张幻灯片上，演示文稿的标题都应始终显示在幻灯片平台的上方或上方。

- 标题附近应包含一些文本/图标/图形/按钮，用户点击后即可调出编辑演示文稿标题的模式。

- 演示文稿屏幕上应提供允许用户更新演示文稿缩略图的途径。

#### 2.2.5. 创建幻灯片并在幻灯片之间移动

- 访问特定幻灯片时，应显示一个按钮，允许用户创建新幻灯片。
- 创建新幻灯片将在幻灯片组末尾添加另一张幻灯片。
- 当幻灯片组至少有两张幻灯片时，控件应出现在幻灯片组内的合理位置：
- 这些控件应为左右两个箭头。
- 用户点击这些箭头时，会跳转到下一张或上一张幻灯片。
- 用户点击相应的键盘按键（在本例中为**左键**和**右键**）时，应发生相同的相应操作。
- 如果用户正在查看第一张幻灯片，则应禁用上一张幻灯片的箭头，并在视觉上有所不同。
- 如果用户正在查看最后一张幻灯片，则应禁用下一张幻灯片的箭头，并在视觉上有所不同。

#### 2.2.6.删除幻灯片

- 访问特定幻灯片时，应显示一个按钮，允许用户删除该幻灯片。
- 如果用户尝试删除幻灯片组中唯一的一张幻灯片，则应弹出一条错误消息，而不是要求删除演示文稿。

注意：删除当前幻灯片后的行为可以完全由您的设计实现。例如：_将用户重定向到上一张幻灯片_

#### 2.2.7. 幻灯片编号

- 查看特定幻灯片时，幻灯片编号应在幻灯片内可见，位于**左下角**。字体大小应为任意颜色的 `1em`，并且应仅在 `50px` x `50px` 的区域内可见。当只剩下一张幻灯片时，此编号将仅显示为“1”。

### 2.3. 功能集 3. 将元素添加到幻灯片

- 当用户被提示输入以下元素的“尺寸”时，尺寸始终以百分比 (%) 表示，数值范围为 0 到 100，其中：
- 对于宽度，100 表示幻灯片的完整宽度，50 表示宽度的一半，以此类推。
- 对于高度，100 表示幻灯片的完整高度，50 表示高度的一半，以此类推。
- 任何元素首次添加到幻灯片时，其位置始终位于幻灯片的左上角。
- 双击幻灯片中的任意元素（0.5 秒内）即可编辑创建此元素时设置的初始属性（将在后续讨论），以及一个名为 _position_ 的额外属性，该属性指定元素左上角在幻灯片上的显示位置。此属性表示为介于 0 到 100 之间的 x 和 y 坐标（类似于上面描述的）。
- 您可以通过将最新创建的元素置于前一个元素之上来调整每个元素的“layer”属性。这在元素彼此叠加的情况下非常有用。
- 右键单击​​幻灯片中每个元素块内的任意位置即可删除它们。

#### 2.3.1. 在幻灯片上添加文本

- 在幻灯片编辑屏幕的某个位置，对于每张幻灯片，都应该有一个明确描述为向当前幻灯片添加文本框的操作。此操作可以立即显示在工具列表中，也可以通过某种可折叠面板隐藏。
- 点击此操作后，应出现一个模态框，并接受用户输入以下信息：
1. 文本区域的大小
2. 文本区域中的文本
3. 文本的字体大小（以“em”为单位，以十进制表示）
4. 文本颜色（以[十六进制颜色代码](https://www.w3schools.com/css/css_colors_hex.asp)表示）。
- 文本始终自上而下左对齐。
- 如果文本溢出，可直接截断。
- 每个文本块的外部都应带有柔和的灰色边框。
- 双击文本块即可编辑上述属性。

#### 2.3.2. 在幻灯片上添加图片

- 在幻灯片编辑屏幕的某处，对于每张幻灯片，都应有一个操作，明确描述为向当前幻灯片添加图片。此操作可以立即显示在工具列表中，也可以通过某种可折叠面板隐藏。
- 点击此操作时，应出现一个模态窗口，接受用户的以下输入：
1. 图片区域的大小
2. 解析 URL 或本地文件，将其解析为整张图片的 base64 字符串编码
3. 为 `alt` 标签添加图片描述
- 双击图片块，用户可以编辑上述属性

#### 2.3.3. 在幻灯片上添加视频

- 在幻灯片编辑屏幕的某处，对于每张幻灯片，都应有一个操作，明确描述为向当前幻灯片添加视频。此操作可以立即显示在工具列表中，也可以通过某种可折叠面板隐藏。
- 点击此操作后，应出现一个模态框，并接受用户输入以下信息：
1. 视频区域的大小
2. 要显示的 YouTube 视频的 URL - 嵌入视频 URL：例如：https://www.youtube.com/embed/dQw4w9WgXcQ?si=ZVLBiX_k2dqcfdBt
3. 视频是否自动播放
- 双击视频块将允许用户编辑上述属性

注意：您可以使用 `<video />` 元素包裹一个容器，以允许用户编辑属性（例如，带有可点击边框），容器的具体 UI 由您决定。

#### 2.3.4. 在幻灯片上添加代码

- 在幻灯片编辑屏幕的某个位置，对于每张幻灯片，都应该有一个操作，明确描述为向当前幻灯片添加代码块。代码块可以类似于 `textarea` 的方式呈现。此操作可以立即显示在工具列表中，也可以通过某种可折叠面板隐藏。每个代码块仅包含一种编程语言。
- 点击此操作时，应出现一个模态窗口，并接受用户输入以下信息：
1. 代码块的大小
2. 代码块中的代码
3. 文本的字体大小（以“em”为单位，小数部分）
- 输入的代码在屏幕上显示时应保留空格
- 如果您完成了“2.4.1”版本，则代码的“font-family”不应更改
- 代码还应根据所选语言进行语法高亮：

- 有效的语言包括 C、Python 和 JavaScript。您需要支持所有这些语言才能获得此功能的分数。
- 此元素应能够根据输入**自动**区分不同的编程语言

- 双击代码块将允许用户编辑上述属性

#### 2.3.5. 🙉🙉🙉 使元素可移动

- 对于 `2.3.1`、`2.3.2`、`2.3.3`、`2.3.4` 的所有版本，请进行以下更改：
- 双击某个块时，位置不再是编辑其位置的选项。
- 单击某个块一次后，其四个角上应各有一个 `5px` x `5px` 的小实心框，因此：
- 如果用户点击并拖动该框，则可以更改框的位置（保持宽高比）。
- 块的任何角都不能超出幻灯片的边缘。

#### 2.3.6. 🙉🙉🙉 使元素可调整大小

- 对于 `2.3.1`、`2.3.2`、`2.3.3`、`2.3.4` 和 `2.3.5` 所有版本，请进行以下更改：

- 双击某个区块时，大小不再是编辑区块大小的选项
- 单击某个区块一次后，其四个角上应各有一个 `5px` x `5px` 的小实心框，从而：

- 如果用户点击并拖动角，则可以增大或减小框的大小。
- 区块的大小调整不能小于宽度或高度的 `1%`。
- 区块的任何角都不能超出幻灯片的边缘。

### 2.4.功能集 4. 更多功能 (13%)

#### 2.4.1. 字体调整

- 在幻灯片编辑屏幕上，用户应该能够更改幻灯片上所有文本框的全局“字体系列”，并且至少可以从 3 种不同的字体系列中进行选择。

#### 2.4.2. 主题和背景选择器

- 应该有一个按钮，该按钮在所有幻灯片上都可见，当用户点击该按钮时，会弹出一个模态框。
- 在此模态框中，您可以指定：

- 当前幻灯片的背景样式。选项包括：
- 纯色
- 渐变
- 图片（覆盖整个幻灯片）
- 根据用户在背景选项中选择的内容，设置默认背景颜色/渐变/图片。
- 这是默认幻灯片背景，而不是白色。
- 当用户当前幻灯片背景样式发生变化时，它将覆盖默认幻灯片背景。

注意：您可以自由选择不同的渐变方向（例如，从上到下/从左到右）。您可以完全自行设计一个用户界面，让用户选择不同的背景选项、颜色/图像。

#### 2.4.3. 预览视图

- 每个幻灯片组都应在某个位置（直接可见或面板后方）有一个按钮，用户可点击该按钮预览演示文稿。
- 预览演示文稿只需在浏览器中打开另一个标签页，其中：
- 幻灯片组在浏览器中全屏显示。
- 箭头控件和幻灯片编号仍然可见且可用，点击箭头应相应地显示上一张/下一张幻灯片。
- 每个区块周围不应有边框。

#### 2.4.4. URL 更新

- 无论是编辑幻灯片还是预览演示文稿，在特定幻灯片上，幻灯片编号都应反映在 URL 中，以便页面刷新时，当前用户能够导航到同一页面。

#### 2.4.5. 🙉🙉🙉幻灯片切换

- 在幻灯片组幻灯片切换时，至少添加一种动画效果。例如：
- 左右滑动
- 淡入淡出或交叉淡入淡出

#### 2.4.6. 🙉🙉🙉幻灯片重新排列

- 每个幻灯片组都应有一个按钮（可以直接点击，也可以在控制面板后面点击）来打开幻灯片重新排列界面。
- 幻灯片重新排列界面应将每张幻灯片显示为一个矩形，每个幻灯片内部都有一个数字，用于指示该幻灯片在所有幻灯片中的索引。
- 矩形的大小应使其能够完全适应视口（假设幻灯片数量少于 10 张）。
- 用户可以点击并拖动特定幻灯片，将其拖放到另外两张幻灯片之间进行重新排列。
- 有一个关闭按钮可以退出此界面。

#### 2.4.7. 🙉🙉🙉修订历史记录

- 每个幻灯片组上都应有一个可访问的按钮（可立即访问，也可在控制面板后访问），用于打开版本历史记录页面。
- 该按钮应显示历史记录列表，以便用户“恢复”，将幻灯片组中的所有幻灯片恢复到先前的状态。
- 每次对幻灯片组进行修改时，都应记录这些先前的状态，且每次保存之间至少间隔 1 分钟。

### 2.5. 代码检查

- 必须在 `frontend` 文件夹中运行 `npm run lint` 命令来运行代码检查。为了获得代码检查部分的分数，你必须确保代码检查不会产生**任何**错误和警告。
